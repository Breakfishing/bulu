- 첫 모달에서 어떤 마커를 추가할 지 선택하고 포인트 마커라면 다음 모달에서 현재 위치 정보를 카카오api를 활용해 위경도를 도로명 주소로 변환하여 노출하고 포인트 이름 입력, 카테고리 선택, 주변 편의시설 유무 설정, 주차장 설정(없음, 무료, 유료), 메모 기능(300자 내외)
- 생성된 마커를 클릭하면 아래에서 위로 등장하는 모달창이 뜨고 해당 포인트의 설정 정보 노출과 함께  해양수산부 국립해양조사원_ROMS 수치예측모델api, 오픈웨더api로 날씨, 기온, 강수, 풍속, 풍향, 파고 수온 정보를 시간 기준 타임테이블로 띄우고 좌우 스크롤로 72시간 까지 데이터 확인
- 타임테이블 하단에는 일출 일몰 시간, 몇물(8물때식) 간조 만조 표시 (가능하면 스크롤로 저점과 고점에 점을찍어 해당 시간에 표시)
- 레이어 전환 아이콘 하단에 화장실 아이콘추가 (지도내 공중화장실 위치 표시하고 추후 직접 추가수정 가능하도록)
- 첫 모달에서 공중화장실 마커를 선택했다면 두번째 모달에서 카카오api를 통해 도로명 주소를 가져와 표시하고 하단에는 관리 상태 별점(최대 5점) 표시.
- 공중화장실 마커를 클릭하면 도로명 주소와 관리 상태 점수 확인 가능하고 하단에는 닫기, 편집, 삭제 버튼으로 관리.
- 폴리곤 기능으로 낚시금지구역을 그릴 수 있고 수정, 삭제 db로 보관.
- 폴리곤을 다 그리면 낚시금지구역(빨간색), 육상항만구역(노란색), 출입불가구역(회색),낚시허용구역(녹색)을 선택하는 모달 추가.

- 폴리곤을 클릭하면 해당 구역에 대한 법률과 벌금 정보를 알려주고 닫기, 편집(구역수정), 삭제 버튼으로 관리.

1. 요청 횟수 제한(카운팅) 문제 해결: "데이터 로컬 캐싱"
기상청과 해양조사원 API는 하루 호출 제한이 엄격합니다. 매번 클릭할 때마다 API를 호출하는 것은 매우 위험합니다.

전략: 타임스탬프 기반 캐싱

localStorage에 데이터를 key(위치+날짜) + data(결과값) + timestamp(조회시각) 형태로 저장합니다.

클릭 시: localStorage에 저장된 정보가 있고, 해당 데이터가 3시간 이내에 조회된 것이라면 API 호출 없이 즉시 그 데이터를 불러옵니다.

없다면: API를 호출하고 데이터를 받아와서 localStorage에 새로 저장합니다.

효과: 동일 지점을 여러 번 확인해도 3시간 동안은 API 요청 횟수가 '1회'로 고정됩니다.

2. 서버 부하 및 UX 문제 해결: "병렬 처리와 낙관적 UI"
계단식(Waterfall)으로 API를 호출하면 대기 시간이 길어지고, 응답이 늦어질 때 UI가 텅 비어 보이게 됩니다.

전략: Promise.all + 로딩 상태 관리

두 API 호출을 동시에 실행(Promise.all)하여 응답 속도를 최대 50%까지 줄입니다.

효과: 로딩 스피너 하나로 두 통신을 통합 관리할 수 있어 CPU나 메모리 부하를 방지합니다.
